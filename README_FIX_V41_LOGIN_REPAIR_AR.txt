إصلاح v41 - معالجة تعليق تسجيل الدخول بعد تنظيف صورة الموظف

سبب المشكلة في v40:
كان هناك استبدال مباشر لدوال أساسية بنفس الاسم مثل updateTopbarUser و hydrateAttachmentImages بطريقة تسببت في تحميل الدالة الجديدة قبل اكتمال تهيئة الصفحة، وفي حالة updateTopbarUser تسبب ذلك في استدعاء ذاتي متكرر قد يعلق صفحة تسجيل الدخول أو يمنع اكتمال الاتصال بقاعدة البيانات.

ما تم إصلاحه:
1. منع الاستبدال المبكر للدوال الأساسية أثناء تحميل JavaScript.
2. إصلاح updateTopbarUser حتى لا يستدعي نفسه بشكل متكرر.
3. الإبقاء على إصلاح صورة الموظف لكن بعد اكتمال تعريف الدوال الأصلية.
4. عدم تعديل إعدادات Supabase أو ملفات SQL.
5. عدم تغيير التصميم.

الملف المعدل:
- app.js

لا توجد أوامر SQL مطلوبة.
