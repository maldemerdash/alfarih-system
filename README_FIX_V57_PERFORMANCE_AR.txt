تعديل v57 لتخفيف ثقل الموقع والاتصال بقاعدة البيانات

ما تم تعديله:
1) لم يتم تغيير التصميم أو منطق الموظفين أو الصلاحيات أو المالية.
2) تم تقليل الحفظ المتكرر إلى Supabase أثناء الكتابة والتنقل داخل الموقع.
3) تم منع تكدس أكثر من عملية حفظ سحابية في نفس الوقت.
4) تم إضافة كاش مؤقت لبيانات المرفقات وروابط التحميل حتى لا يتم طلب نفس المرفق من قاعدة البيانات والتخزين أكثر من مرة.
5) تم إضافة ملف SQL اختياري مهم لتحسين أداء الجداول المستخدمة في تسجيل الدخول والحفظ والمرفقات.

الملفات التي ترفعها على الموقع:
- app.js
- index.html إذا كنت ترفع المشروع كاملًا كما هو
- style.css إذا كنت ترفع المشروع كاملًا كما هو
- باقي الملفات كما هي عند رفع النسخة كاملة

المطلوب في Supabase:
افتح SQL Editor وشغل ملف:
SUPABASE_PERFORMANCE_INDEXES_V57.sql

ملاحظة مهمة:
هذا ليس تحويلًا كاملًا إلى Next.js؛ لأن النسخة الحالية موقع ثابت كبير بملف app.js واحد، والتحويل الكامل دفعة واحدة قد يكسر المنطق الحالي. هذا التعديل هو تحسين آمن داخل نفس النسخة لتخفيف الثقل فورًا.
