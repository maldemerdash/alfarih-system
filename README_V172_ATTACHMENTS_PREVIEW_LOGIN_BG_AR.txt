إصلاحات v172 - منطق المرفقات والمعاينة وخلفية الدخول
=====================================================

المشاكل المطلوبة:

- إظهار حالة واضحة عند رفع أي مرفق في الموقع بدل بقاء الزر ساكنا.
- إضافة علامة عين لمعاينة المرفق داخل نافذة منبثقة من هوية الموقع.
- تحديث الواجهة بعد عمليات الحفظ بدون إبطاء الموقع.
- منع ظهور خلفية تسجيل الدخول الافتراضية قبل تحميل الخلفية المختارة.

ما تم تنفيذه:

1. إضافة طبقة موحدة فوق خانات الملفات `input[type=file]`:
   - عند اختيار ملف تظهر حالة `جاري الرفع`.
   - بعد نجاح الحفظ تعود الخانة إلى شكلها الحالي مع حالة `تم الإرفاق`.
   - توجد مهلة أمان لإزالة حالة الرفع إذا تأخر المتصفح أو فشل حدث الحفظ.

2. إضافة معاينة مرفقات موحدة:
   - أزرار فتح المرفقات تتحول إلى زر عين.
   - الضغط يفتح نافذة داخلية باسم `attachmentPreviewModal`.
   - الصور و PDF تظهر داخل النافذة.
   - الأنواع الأخرى تعرض رسالة مناسبة مع زر تحميل.

3. تحديث الواجهة بعد الحفظ:
   - بعد اكتمال الحفظ يتم تحديث الصفحة النشطة فقط.
   - لا يتم تشغيل حفظ إضافي بعد الحفظ، للحفاظ على السرعة ومنع الحلقات المتكررة.

4. تحسين بداية شاشة الدخول:
   - يتم قراءة خلفية الدخول المحفوظة مبكرا من إعدادات المنشأة أو الكاش المحلي.
   - عند وجود خلفية محفوظة لا تظهر الخلفية الافتراضية قبلها.
