نسخة v136 - إكمال برمجيات الإعدادات بالترتيب

تم البناء على نسخة v135 التي ثبت نجاح ربط صلاحيات المدير، ولم يتم تغيير منطق الصلاحيات العامل.

التعديلات المنفذة:
1) الصلاحيات والأمان:
- تثبيت شاشة v135 كالشاشة المعتمدة.
- تعطيل الواجهات القديمة بصريًا حتى لا تتداخل مع الحفظ الحالي.
- الإبقاء على منطق: الموظف -> حساب المستخدم المرتبط -> الصلاحيات.

2) الحساب الشخصي:
- تحويل القسم من بطاقة مؤجلة إلى شاشة فعلية.
- عرض الاسم، البريد، الصلاحية، حالة الحساب، والموظف المرتبط.
- السماح بتعديل الاسم الظاهر فقط.
- منع تعديل البريد والصلاحية من الحساب الشخصي لأنها تدار من إدارة المستخدمين.

3) الإشعارات:
- تحويل القسم إلى شاشة إعدادات فعلية.
- إضافة تنبيهات: العقد، الهوية/الإقامة، الجواز، رخصة العمل، الوثائق، فترة الإشعار، الغياب، الإجازات والسفر.
- تحديد عدد الأيام قبل التنبيه.
- حفظ الإعدادات ضمن حالة النظام.

4) الأقسام / الهيكل الإداري:
- إضافة شاشة داخل الإعدادات باسم الأقسام.
- إضافة إدارة مع مدير إدارة.
- إضافة قسم تابع لإدارة مع مدير قسم.
- إضافة مهنة تابعة لقسم.
- ربط شاشة الموظف بحيث اختيار الإدارة يعرض أقسامها فقط، واختيار القسم يعرض المهن التابعة، والمدير المباشر يظهر تلقائيًا.

5) إعداد العمل:
- إضافة بطاقة مراجعة تظهر عدد الفترات وأيام العمل المفعلة والتنبيه عند وجود تعارض في الفترات.

الملف المعدل الأساسي:
- app.js

SQL:
- لا يوجد SQL جديد مطلوب لهذه النسخة.
- تم الاعتماد على الجداول الموجودة، خصوصًا app_user_profiles للصلاحيات والحسابات.

ملاحظة:
- إعدادات الإشعارات والهيكل الإداري تحفظ ضمن حالة النظام السحابية إذا كان حفظ Supabase مفعلًا، مع وجود نسخة محلية في المتصفح كاحتياط.
