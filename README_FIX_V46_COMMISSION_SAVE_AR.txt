إصلاح v46 - حفظ العمولة فور إصدار المخالصة

تم تعديل منطق إصدار مخالصة العمولة فقط، بدون تعديل تصميم الواجهة أو منطق الصور أو الصلاحيات.

ما تم تعديله:
1. عند الضغط على إصدار المخالصة يتم حفظ العمولة مباشرة داخل سجل الموظف.
2. يتم تحديث بداية استحقاق العمولة التالية مباشرة بعد الحفظ.
3. يتم حفظ السجل في IndexedDB ثم مزامنته مع Supabase عبر saveCloudStateNow.
4. لا يتم اعتماد المخالصة إذا فشل الحفظ.
5. تم منع الضغط المزدوج على زر إصدار المخالصة حتى لا تتكرر العمولة.
6. تم منع إصدار مخالصة لموظف غير محفوظ؛ تظهر رسالة تطلب حفظ الموظف أولا.

الملف الأساسي المعدل:
app.js

لا توجد أكواد SQL مطلوبة.
