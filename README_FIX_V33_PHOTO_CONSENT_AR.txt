تعديل v33 - صورة الموظف ومرفق إقرار التوثيق

الملفات المعدلة:
- app.js
- style.css

ما تم إصلاحه:
1) صورة الموظف:
- رفع الصورة يتطلب وجود رقم الهوية والجوال حتى يتم ربطها برقم الموظف الصحيح.
- تظهر الصورة مباشرة داخل نموذج الموظف بعد الرفع.
- بعد حفظ الموظف تظهر فوراً في جدول الموظفين باستخدام كاش محلي للرابط بدون انتظار توليد رابط جديد.
- إذا كان المستخدم مربوطاً بموظف، تظهر صورة الموظف في رأس الموقع.
- عند تسجيل الخروج والدخول تبقى الصورة لأنها محفوظة في Supabase ومربوطة بسجل الموظف.
- إذا حذفت/اختفت الصورة، يرجع رأس الموقع للحرف الأول بدل فراغ أو وميض.

2) مرفق إقرار التوثيق:
- لا يمكن إصدار الإقرار إلا بعد رفع مرفق فعلي بنجاح.
- تظهر حالة "تم الإرفاق" داخل نافذة الإقرار.
- يظهر زر عرض المرفق داخل نافذة الإقرار بعد الرفع.
- إذا فشل الرفع تظهر رسالة منبثقة واضحة ولا يتم إصدار الإقرار.
- إذا كانت الهوية أو الجوال غير مكتملة يمنع الرفع حتى لا يرتبط المرفق بموظف خاطئ.

لا يوجد SQL مطلوب.
