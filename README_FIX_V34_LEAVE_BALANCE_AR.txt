# v34 - سياسات رصيد الإجازات واعتماد الرصيد السالب

تمت إضافة منطق رصيد الإجازات بدون تغيير الهوية البصرية العامة:

- إضافة قسم جديد داخل الإعدادات باسم: سياسات الإجازات والرصيد.
- التحكم في 21 يوم قبل 5 سنوات و30 يوم بعد 5 سنوات.
- التحكم في طريقة الاحتساب: تراكمي / سنوي / يدوي.
- التحكم في احتساب الأيام: كل الأيام أو أيام العمل فقط.
- استبعاد العطل الرسمية عند تفعيلها.
- السماح بالرصيد السالب بصلاحية اعتماد.
- عرض رصيد الموظف داخل نافذة طلب الإجازة قبل الإرسال.
- عند نقص الرصيد تظهر نافذة منبثقة مع الرصيد المتوقع.
- صاحب الصلاحية يمكنه اعتماد الطلب بالرغم من الرصيد السالب.
- التعارض الزمني مع إجازة أو سفر ما زال يمنع الطلب ولا يمكن تجاوزه.
- الخصم النهائي يتم بعد تسجيل مباشرة الموظف من الإجازة لأن الرصيد المستخدم يعتمد على returnDate/returnConfirmedAt.
- إضافة ملخص مختصر داخل بطاقة الإجازات الموجودة في الشاشة الرئيسية، بدون إضافة بطاقة كبيرة جديدة.

- style.css
