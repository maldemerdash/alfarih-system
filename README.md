# نواة - نظام إدارة الموظفين

تطبيق عربي متجاوب لإدارة الموظفين والموارد البشرية، يعمل مباشرة في المتصفح
ولا يحتاج إلى تثبيت مكتبات.

## بنية التطبيق

تم توحيد المشروع الأصلي، الذي كان موزعًا على خمس صفحات وعدة ملفات تنسيق
وبرمجة، في تطبيق صفحة واحدة:

- `index.html`: هيكل الواجهة وجميع الشاشات والنوافذ.
- `style.css`: نظام التصميم والاستجابة للشاشات المختلفة.
- `app.js`: منطق التطبيق والبيانات والتفاعلات.

## التشغيل

يمكن فتح `index.html` مباشرة، أو تشغيل خادم محلي:

```bash
python3 -m http.server 4173
```

ثم زيارة `http://localhost:4173`.

## المزايا الحالية

- لوحة مؤشرات للموظفين والحضور والإجازات والرواتب.
- إضافة الموظفين وتعديلهم وحذفهم والبحث والتصفية.
- ملف موظف شامل للبيانات الشخصية والوظيفية والهوية والجوازات والاتصال.
- رقم موظف تلقائي من آخر رقمين للهوية وآخر رقمين للجوال بصيغة `34-78`.
- حساب تلقائي للعمر ومدة الخدمة والعقد والراتب والعمولة.
- حسابات بنكية وجوازات ووثائق وملاحظات متعددة مع المرفقات.
- عرض جدولي أو بطاقات مع تصدير CSV.
- واجهة عربية متجاوبة للجوال.

## حفظ البيانات

- تُحفظ بيانات الموظفين والمرفقات محليًا في `IndexedDB`.
- تُحفظ الإجازات والمسميات الوظيفية في `localStorage`.

## ملاحظات التحويل

صفحة تسجيل الدخول ونظام الصلاحيات الموجودان في المشروع الأصلي غير مفعّلين في
النسخة الموحدة الحالية. الربط بحسابات فعلية أو قاعدة بيانات مركزية يحتاج إلى
خدمة خلفية عند تجهيز التطبيق للنشر.

## تحديث: إنشاء المستخدم من داخل الموقع

تمت إضافة حقل كلمة المرور في شاشة إدارة المستخدمين. عند إضافة مستخدم جديد، يستدعي الموقع دالة Supabase Edge Function باسم `admin-create-user` لإنشاء حساب الدخول في Supabase Authentication ثم حفظ الصلاحية في `app_user_profiles`.

قبل استخدام الزر يجب نشر الدالة الموجودة في:

`supabase/functions/admin-create-user/index.ts`

راجع ملف:

`SUPABASE_EDGE_FUNCTION_ADMIN_CREATE_USER.md`
