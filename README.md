# نسخة v165 - إصلاح ثبات المرفقات

تم البناء على نسخة v164 مع إبقاء بنية التشغيل نفسها:

- `index.html`
- `style.css`
- `app.js`
- ملفات Supabase وملفات الشرح المرافقة

أهم ما تم تعديله في v165:

- إصلاح اختفاء المرفقات بعد تسجيل الخروج والدخول مرة أخرى.
- إضافة فهرس دائم لبيانات المرفقات داخل حالة الموقع السحابية حتى لا يعتمد العرض على ذاكرة جلسة المتصفح فقط.
- توحيد قراءة المرفقات من جدول `attachments` مع fallback من الفهرس المحلي/السحابي عند تعذر قراءة الجدول بسبب سياسات Supabase.
- منع كود ملف الموظف من مسح أرقام المرفقات من السجل إذا تعذر إنشاء رابط عرض مؤقت.
- دعم مرفقات الهوية، الصورة، التوقيع، البصمة، العقد، الجوازات، الآيبان، وثائق الموظف، الإقرار، شعار المنشأة، خلفية الدخول، العنوان الوطني، ووثائق المنشأة.
- إضافة ملف `SUPABASE_ATTACHMENTS_STORAGE_SQL.sql` لإنشاء جدول `attachments` و buckets وسياسات Storage المطلوبة.
- إضافة أداة فحص من الكونسول باسم `nawahAttachmentDiagnostics()` لمعرفة هل المرفق محفوظ ومتاح للعرض أم لا.
- تحديث روابط الكاش إلى `v=165`.

ملاحظات مهمة:

- إذا كانت المرفقات القديمة موجودة في Storage لكن لا تظهر، فالأغلب أن المشكلة كانت في قراءة جدول `attachments` أو سياسات Storage، وليس في الملف نفسه.
- يجب تشغيل `SUPABASE_ATTACHMENTS_STORAGE_SQL.sql` مرة واحدة من Supabase SQL Editor لضمان الحفظ الحقيقي والقراءة بعد تسجيل الدخول.
- تم تجنب تفكيك `app.js` إلى ملفات متعددة لأن الملف يحتوي سلسلة تصحيحات تعتمد على ترتيب التنفيذ.
- لا يتم حذف أي بيانات من المتصفح أو Supabase بمجرد استبدال الملفات.
- ملفات SQL الاختيارية التي تحذف أو تصفر بيانات مالية ما زالت اختيارية ولا تشغل إلا عند الحاجة.
