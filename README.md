# نسخة v166 - التنبيهات الخاصة

تم البناء على نسخة v165 مع إبقاء بنية التشغيل نفسها:

- `index.html`
- `style.css`
- `app.js`
- ملفات Supabase وملفات الشرح المرافقة

أهم ما تم تعديله في v166:

- إضافة صفحة جديدة باسم `التنبيهات الخاصة` أسفل `وثائق المنشأة`.
- إضافة جدول للتنبيهات الخاصة وزر `إضافة تنبيه خاص`.
- إضافة نافذة منبثقة لإنشاء التنبيه تشمل: الموضوع، النص، تاريخ الإنشاء، تاريخ التذكير، التذكير قبل، والربط بموظف اختياري.
- عند اختيار الربط بموظف تظهر قائمة الموظفين الحاليين.
- إضافة إجراءات لكل تنبيه: إنهاء، تأجيل، حذف، مع ألوان مناسبة لهوية الموقع.
- عند التأجيل يتم طلب تاريخ تذكير جديد، ويصبح هو التاريخ المعتمد بدل التاريخ القديم.
- إضافة أيقونة مستقلة أعلى الموقع للتنبيهات الخاصة بجوار جرس الإشعارات وصورة الحساب.
- قائمة الأيقونة العلوية تستخدم نفس تصميم قائمة الإشعارات الحالية وتعرض التنبيهات التي دخلت فترة التذكير فقط.
- حفظ التنبيهات الخاصة ضمن حالة الموقع السحابية باسم `privateAlertsV166`.
- تحديث روابط الكاش إلى `v=166`.

ملاحظات مهمة:

- التنبيه يظهر في الأيقونة العلوية عندما يصبح تاريخ التذكير داخل مدة `التذكير قبل`، ويظل ظاهرًا إذا تأخر حتى يتم إنهاؤه أو تأجيله أو حذفه.
- إذا كانت المرفقات القديمة موجودة في Storage لكن لا تظهر، فالأغلب أن المشكلة كانت في قراءة جدول `attachments` أو سياسات Storage، وليس في الملف نفسه.
- يجب تشغيل `SUPABASE_ATTACHMENTS_STORAGE_SQL.sql` مرة واحدة من Supabase SQL Editor لضمان الحفظ الحقيقي والقراءة بعد تسجيل الدخول.
- تم تجنب تفكيك `app.js` إلى ملفات متعددة لأن الملف يحتوي سلسلة تصحيحات تعتمد على ترتيب التنفيذ.
- لا يتم حذف أي بيانات من المتصفح أو Supabase بمجرد استبدال الملفات.
- ملفات SQL الاختيارية التي تحذف أو تصفر بيانات مالية ما زالت اختيارية ولا تشغل إلا عند الحاجة.
