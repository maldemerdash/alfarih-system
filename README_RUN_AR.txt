طريقة تشغيل النسخة v177
=======================

هذه النسخة تحتوي على ملفات الواجهة الأمامية وملفات Supabase المرافقة:
- index.html
- style.css
- app.js
- sar-symbol.png
- supabase/functions/admin-create-user/index.ts
- ملفات SQL وملفات README الخاصة بالإصدارات السابقة

التشغيل السريع:
1) افتح index.html مباشرة في المتصفح.

أو شغل خادما محليا من داخل المجلد:
python3 -m http.server 4173

ثم افتح:
http://localhost:4173

ملاحظات:
- البيانات المحلية تحفظ في IndexedDB و localStorage.
- النسخة تحتوي إعدادات Supabase وتسجيل دخول وإدارة مستخدمين عند تجهيز ملفات SQL و Edge Function.
- لتحديث صلاحيات المستخدمين من الموقع، انشر Edge Function الموجودة في:
  supabase/functions/admin-create-user/index.ts
- شغل ملفات SQL المطلوبة في Supabase SQL Editor حسب حالة قاعدة البيانات لديك.
- للمرفقات تحديدا شغل الملف التالي مرة واحدة:
  SUPABASE_ATTACHMENTS_STORAGE_SQL.sql
  هذا الملف ينشئ جدول attachments و buckets وسياسات Storage حتى تبقى المرفقات ظاهرة بعد تسجيل الخروج والدخول.
- خط Almarai من Google Fonts يحتاج اتصالا بالإنترنت؛ بدون الإنترنت سيستخدم المتصفح خطا بديلا.
- تم اختبار منطق المرفقات مع سيناريو إعادة التحميل وفشل قراءة جدول attachments للتأكد من عدم مسح أرقام المرفقات من السجلات.
- تم اختبار التنبيهات الخاصة: الإضافة، الربط بموظف، الظهور في الأيقونة العلوية، التأجيل بتاريخ جديد، الإنهاء، الحذف، والحفظ ضمن حالة الموقع.
- في v167 تم اختبار إصلاحات التنبيهات الخاصة: إخفاء أزرار الإنهاء والتأجيل بعد الإنهاء، جدول المحذوفات، تثبيت ترتيب الأيقونات العلوية، وتسجيل التنبيه المرتبط داخل ملاحظات الموظف.
- في v168 تم اختبار تسريع فتح تحرير الموظف وقائمة الموظفين: فتح نافذة التحرير أصبح فوريًا تقريبًا، وقائمة الموظفين لم تعد تبني الجدول والبطاقات معًا في نفس اللحظة.
- في v169 تم اختبار إغلاق قوائم الإشعارات المتبادلة، واختبار حفظ تعديل الموظف بنفس رقم الهوية دون ظهور رسالة تكرار الهوية لنفس السجل.
- في v170 تم تحويل تأكيد حذف التنبيه الخاص من رسالة المتصفح الافتراضية إلى نافذة منبثقة داخلية بهوية الموقع.
- في v171 تم تحويل تأكيد حذف السلفة إلى نافذة داخلية، وإضافة ستار تحميل يمنع ظهور الرسائل المؤقتة والتصميم القديم أثناء بداية تحميل الصفحة.
- في v172 تم توحيد حالة رفع المرفقات، وإضافة معاينة بعلامة العين داخل نافذة من هوية الموقع، وتحديث الواجهة بعد الحفظ دون حفظ متكرر، وتجهيز خلفية الدخول قبل عرض شاشة الدخول.
- في v173 تم تخفيف جلب المرفقات بإضافة كاش وإزالة مراقبات زائدة، وتحويل شاشة تجهيز النظام إلى خلفية بيضاء فقط، ومنع أي تدرج لوني في خلفية تسجيل الدخول.
- في v174 تم تطبيق معاينة مرفقات السفر بعلامة العين للتذكرة والتأشيرة داخل صفحة الإجازات والسفر وسجل الموظف، مع عدم جلب الملفات إلا عند المعاينة.
- في v175 أصبحت بطاقة الشاشة الرئيسية `مسير هذا الشهر` تعرض صافي الصرف النهائي للمسير بعد كل الإضافات والخصومات، وليس مجموع الرواتب الأساسية.
- في v176 تم تثبيت نافذة إضافة الإدارة/القسم/المهنة أمام ملف الموظف، وتنظيم إعدادات الأقسام إلى جداول منفصلة للإدارات والأقسام والمهن مع أزرار تعديل وحذف، ومنع حذف العناصر المرتبطة بملفات موظفين.
- في v177 تم توحيد خلفية تسجيل الدخول على مصدر نهائي واحد؛ عند حفظها من إعدادات المنشأة تظهر مباشرة عند تسجيل الدخول، مع مسح كاش الخلفيات القديمة ومنع التدرجات الافتراضية.
