نسخة v65 - حل مصدر البيانات السحابي

المشكلة التي عالجتها هذه النسخة:
- ظهور الموظفين في متصفح واختفاؤهم في متصفح آخر بسبب الاعتماد على التخزين المحلي.
- احتمال حفظ حالة فارغة فوق بيانات موجودة في Supabase.

ما تم تعديله:
- جعل Supabase هو المصدر الأساسي عند فتح الموقع.
- عند وجود app_state في Supabase يتم تحميله أولا حتى لو كان فارغا.
- منع حفظ employees = [] فوق حالة سحابية فيها موظفون.
- عند إنشاء أول بيانات جديدة يتم حفظها في Supabase لتظهر في المتصفحات الأخرى.

لا يوجد SQL إجباري لهذه النسخة إذا كان جدول app_settings موجودا كما ظهر في الفحص.

ملفات الرفع على GitHub Pages:
- app.js
- index.html

بعد الرفع استخدم تحديث قوي للمتصفح.
